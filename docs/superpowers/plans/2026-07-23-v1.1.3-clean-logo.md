# v1.1.3 Clean Logo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an opt-in `Clean Logo` profile that accepts only exact Rockstar-rendered, source-supported, locally non-regressing primitive layers and produces zero unsupported transparency or edge violations on the supplied eye logo.

**Architecture:** Keep all existing profiles and their fitting paths unchanged. Route only the new `clean-logo` runner through the existing resident CUDA primitive search with fixed opaque paint, then pass its ranked finalists through a CPU sanitizer that snaps colors to real source pixels and evaluates exact anti-aliased Rockstar SVG replays before committing. Finish with a deterministic top-layer removal audit and expose the sanitizer counters in `metrics.json`.

**Tech Stack:** C# 7.3 / .NET Framework 4.8, SkiaSharp 4.150.0, existing resident CUDA scorer protocol, JSON profiles, console check harness.

## Global Constraints

- Code, tests, profiles, and documentation are written in English.
- `v1-clean-logo` is opt-in and is never the default.
- `v1-beam-clean`, `v1-perceptual-alexnet`, and `v1-catalog-quality` retain their current behavior and output hashes.
- Clean Logo uses only rotated ellipse, rotated rectangle, and rotated triangle primitives; it does not use catalog shapes, perceptual reranking, overfit, replacement, or refinement.
- Every committed Clean Logo layer is fully opaque, has a minimum axis of four, uses a real source RGB value, and is accepted from the exact Rockstar-rendered replay.
- No safe improving finalist is a successful early completion, not a switch to another mode.
- Nonzero final support or unsupported-edge violations are fatal and prevent import artifacts from being written.
- Production README, packaging, tags, and releases remain unchanged until the eye-logo visual result is accepted.

### Task 4: Implement Clean Logo proposal sanitation

**Files:**

- Create: `native/GTAEmblemMaker.Core/CleanLogoSanitizer.cs`
- Create: `native/GTAEmblemMaker.Checks/CleanLogoSanitizerChecks.cs`
- Modify: `native/GTAEmblemMaker.Checks/Program.cs`

- [ ] Add deterministic 512×512 synthetic tests for:

  - Allowed alpha support: target alpha `> 0`, dilated by one pixel.
  - Allowed edge support: target neighboring premultiplied RGBA difference `>= 24`, dilated by one pixel.
  - Real-source color snapping: prefer footprint pixels with target alpha `>= 128`, unpremultiply RGB, and select the nearest RGB to the proposal.
  - Transparent support rejection: reject any exact-replay output alpha `>= 16` outside allowed alpha support.
  - Clean error: use target alpha `a` to weight RGB error by `a` and alpha error by `4 - 3a`.
  - Local non-regression: require total clean error to decrease and no 16×16 tile to increase.
  - Safe acceptance: return the snapped state and exact replay buffer.

- [ ] Run the checks. Expected: compile failure because the sanitizer types do not exist.
- [ ] Implement `CleanLogoMetrics` with these counters:

  ```csharp
  public int SupportRejectedLayers { get; internal set; }
  public int LocalRegressionRejectedLayers { get; internal set; }
  public int ColorSnappedLayers { get; internal set; }
  public int EdgeRemovedLayers { get; internal set; }
  public int FinalSupportViolationPixels { get; internal set; }
  public int FinalUnsupportedEdgePixels { get; internal set; }
  ```

- [ ] Implement a single `CleanLogoSanitizer` that owns the target-derived masks and current clean error.
- [ ] Convert resident chain finalists to `FitCandidate`, order by CUDA score with stable input-index ties, and test them in that order.
- [ ] For each finalist:

  1. Render it alone to obtain its exact footprint.
  2. Reject it if the footprint has no visible target pixel.
  3. Snap its RGB to the nearest actual unpremultiplied source color under the footprint, preferring source alpha `>= 128`.
  4. Convert it to a fully opaque `ShapeState`.
  5. Exact-replay it onto the current buffer.
  6. Reject transparent spill where output alpha is at least `16` outside allowed alpha support.
  7. Reject unless full clean error strictly decreases and every 16×16 tile is non-regressing.
  8. Return the first safe improving state and exact replay.

- [ ] Keep the clean error integral and deterministic. Use Q8 weight-map values and a fixed alpha denominator of `255`; checked 64-bit accumulation must cover the worst 512×512 input.
- [ ] Rebuild and rerun all checks.
- [ ] Commit:

  ```powershell
  git add native/GTAEmblemMaker.Core/CleanLogoSanitizer.cs native/GTAEmblemMaker.Checks/CleanLogoSanitizerChecks.cs native/GTAEmblemMaker.Checks/Program.cs
  git commit -m "feat: sanitize clean logo proposals"
  ```
