# v1.1.1 Catalog Quality Parity Repair Plan

> **Status:** Proposed for approval. Do not implement, package, publish, retag, or modify the v1.1.0 release until explicitly authorized.

**Goal:** Repair only the Catalog Quality path so it reproduces the accepted catalog prototype's fitting and export semantics, removes the measured performance regression, and leaves every historical V1 profile and payload byte unchanged.

**Approach:** Keep the existing native application, `FittingEngine`, profile parser, exact exporter, catalog atlas, and LPIPS implementation. Give `v1-catalog-quality` one versioned compatibility runner mode, replace its sequential catalog sidecar with one resident CUDA layer command, and bypass the legacy four-pixel clamp only for verified catalog identities. No second fitting framework is introduced.

## Proven release divergence

| Evidence | Accepted artifact | Published v1.1.0 package |
| --- | ---: | ---: |
| Canonical RGBA SHA-256 | `d0b111d5acc8d671417a035c62293f22f797264f720eeaabcbb5226f459c1914` | `80319d17544318d0ba44647cc3fdf10410b2f08605966aac282d2c91d97eb720` |
| First selected layer | candidate `137052`, ellipse `(280,330,136,137)`, alpha `254`, energy `0.3316215210964145` | candidate `272036`, ellipse `(280,329,137,136)`, alpha `251`, energy `0.331947403365683` |
| Final layers | `1,345` | `1,365` |
| Generated code | `1,249,106` | `1,249,958` |
| Catalog layers | `546` | `522` |
| Selected catalog axes below four | `257` axes in `240` layers | `218` axes in `207` layers before export; none survive payload export |
| Exact payload-render SHA-256 | `8a148d573189e85dfe419119b0a8ba0de5fae3b7a20d8e6d658d212fcaadce0b` | `ab2daf1cdc5159bf503b8916d541502dc678dad524e8e0a3b6d37e35a74fd64d` |
| Full-budget wall time | `1,279,782.7272 ms` | `5,585,578 ms` |

The one differing opaque source pixel does not cause the first-layer mismatch: injecting the accepted canonical RGBA into the published package still selects the published package's first layer. The first behavioral divergence is therefore proposal/search/scorer semantics. A second independent correctness fault occurs in `Shapes.ToExportShape`, where the shared legacy four-pixel clamp rewrites catalog geometry after fitting. The performance regression comes from the sequential 11-identity `CatalogCandidateSearch`, repeated atlas transmission and H2D upload, and full-payload reconstruction for every code-aware finalist.
