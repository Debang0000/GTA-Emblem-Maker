# GTA Emblem Maker v1.1.3 Clean Logo Design

## Goal

Add an opt-in `Clean Logo` mode that prioritizes clean boundaries and source-faithful colors over gradient smoothness or micro-detail. The mode must not introduce visible lines, dark repair fringes, or colors that are absent from the source image.

## Scope

- Add a non-default `v1-clean-logo` profile displayed as `Clean Logo`.
- Add a dedicated `clean-logo` runner so the behavior cannot affect existing runners.
- Preserve the configuration and behavior of `Beam Clean`, `Perceptual AlexNet 224`, and `Official Catalog Quality`.
- Use the supplied transparent eye logo as the first visible acceptance image.
- Keep README, packaging, tags, and release publication unchanged until the visible result is accepted.
- Set the development engine version and the new profile minimum engine version to `1.1.3`.

## Profile

The Clean Logo profile uses:

- a `1,250,000` character payload budget;
- at most `1,500` layers, with earlier completion when no safe improving candidate remains;
- a minimum axis of `4`;
- rotated ellipses, rectangles, and triangles from layer 1;
- no Official Catalog identities;
- fixed fully opaque candidate layers;
- `edge-plus-laplacian` for opaque sources;
- `edge-plus-laplacian-alpha-protect` for transparent sources;
- no perceptual reranker;
- no overfit, replacement, or refinement pass.

The profile loader will allow `perceptualRerank` to be omitted. Existing profiles retain their current required values and runtime behavior.

## Data Flow

1. Load and canonicalize the source through the existing 512 by 512 premultiplied RGBA path.
2. Run the existing resident CUDA primitive search with alpha mutation disabled and alpha fixed to 255.
3. Order the resident chain finalists by their existing score without using Official Catalog masks.
4. Pass the finalists to the Clean Logo sanitizer before committing the layer.
5. Recolor and replay each finalist with the exact Rockstar SVG transform and antialiasing used by `payload-preview.png`.
6. Commit the first safe improving finalist, or complete fitting when the round contains none.
7. Run the final edge audit and build `FitResult`, payload code, previews, and run artifacts only from the sanitized sequence.

## Clean Logo Sanitizer

### Source masks

The sanitizer derives two immutable masks from the canonical target:

- **Allowed alpha support:** pixels with target alpha greater than zero, dilated by one pixel in all directions.
- **Allowed edge support:** pixels whose maximum neighboring premultiplied RGBA channel difference is at least 24, dilated by one pixel in all directions.

The one-pixel dilation permits ordinary output antialiasing without permitting detached marks.

### Exact replay

The sanitizer starts from the same transparent or opaque initial canvas as the fitter. Every proposal is rendered before commit with the same Rockstar export transform, opacity quantization, path, and Skia antialiasing used for the payload preview. The exact sanitized current image is uploaded for the next resident CUDA round, so later candidates are scored against the layers that will actually be exported.

### Color safety

For each layer, the sanitizer finds the nearest real source RGB color among target pixels covered by the layer. It replaces the fitted RGB value with that source color before evaluating the proposal. Source colors are unpremultiplied before comparison. Pixels with target alpha at least 128 are preferred; if the footprint contains none, all pixels with nonzero target alpha are eligible.

A proposal with no visible source pixel in its footprint is rejected.

### Boundary safety

For transparent sources, a proposal is rejected when its exact replay creates any pixel with alpha at least 16 outside the allowed alpha support.

### Local safety

A proposal must reduce the complete clean-logo error and may not increase the error of any 16 by 16 tile.

For target alpha `a` normalized to the range 0 through 1, the per-pixel clean-logo error is:

```text
pixelWeight * (a * (redError² + greenError² + blueError²) + (4 - 3a) * alphaError²)
```

This is the ordinary RGBA squared error at opaque pixels. At fully transparent pixels it ignores meaningless RGB and weights alpha four times, so black paint cannot improve a transparent overshoot.

### Final edge audit

After replay, every output edge with a maximum neighboring premultiplied RGBA channel difference of at least 24 must fall inside the allowed edge support.

If an unsupported edge exists, the sanitizer removes the topmost layer contributing to the violating pixels and replays the retained sequence. Removal repeats until the unsupported edge count is zero. This process always terminates because the empty sequence contains no output edge.

### Early completion

The sanitizer tries the resident chain finalists in score order. If none is both safe and improving, fitting completes normally. The runner never switches to another profile.

## Metrics

`metrics.json` will add a `cleanLogo` object for Clean Logo runs:

```json
{
  "supportRejectedLayers": 0,
  "localRegressionRejectedLayers": 0,
  "colorSnappedLayers": 0,
  "edgeRemovedLayers": 0,
  "finalSupportViolationPixels": 0,
  "finalUnsupportedEdgePixels": 0
}
```

Both final violation counts must be zero. Existing profile metrics remain unchanged.

## UI and Artifacts

- The existing profile dropdown discovers `Clean Logo` from the profile directory.
- Clean Logo is not the default selection.
- The completed output shown in the UI is the sanitized final payload preview.
- `preview.png`, `payload-preview.png`, `shapes.json`, `payload.svg`, `import.js`, `metrics.json`, `profile.json`, and `trace.json` all describe the same sanitized layer sequence.

## Error Handling

- Invalid Clean Logo profile values fail profile loading with `ProfileValidationException`.
- No safe candidate is a successful early completion, not an error.
- A nonzero final support or edge violation is an invariant failure; no import code is produced for that result.
- The runner does not fall through to a legacy mode.

## Verification

### Automated

- Profile loading finds `v1-clean-logo`, keeps `v1-beam-clean` as the sole default, and rejects Clean Logo on engines older than 1.1.3.
- Clean Logo requests disable alpha mutation and fix minimum, maximum, and initial alpha to 255.
- Catalog search is absent from the Clean Logo profile.
- Sanitizer checks cover color snapping, transparent support rejection, tile regression rejection, unsupported-edge removal, and zero-violation termination.
- Final artifacts use the sanitized shapes and exact payload rendering.
- Existing checks and golden outputs for all v1.1.2 profiles continue to pass.
- The x64 application and bundle projects build successfully.

### Visible acceptance

Run the supplied eye logo through `Clean Logo` and `Official Catalog Quality`, then compare their final `payload-preview.png` files beside the canonical target.

Clean Logo is accepted only when:

- the two lower circle-and-eye intersections contain no black fringe;
- the transparent background contains no detached spikes or marks;
- the triangle and circular strokes contain no source-absent color streaks;
- the overall shape remains recognizable and the gradient direction is preserved;
- both final violation metrics are zero;
- generated code remains at or below `1,250,000` characters.

No README update, package, tag, or release occurs before this visible result is reviewed and accepted.
